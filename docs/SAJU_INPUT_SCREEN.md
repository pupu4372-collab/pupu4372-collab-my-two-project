# 사주 입력 화면 디자인 리뉴얼 가이드

## 목적

사주 입력 화면은 사용자가 반려동물의 기본 정보를 입력해 **K-사주 결과**를 받는 핵심 전환 화면이다.

새 디자인의 목표는 다음과 같다.

- 사용자가 어렵게 느낄 수 있는 생년월일시 입력을 쉽고 부드럽게 만든다.
- "출생 시간을 모름"도 자연스럽게 선택할 수 있게 한다.
- 입력 중 이탈을 줄이고, 결과 확인까지 빠르게 이어지게 한다.
- 결과 이후 별자리 운세와 펫·집사 궁합으로 자연스럽게 확장한다.
- 모바일 앱(WebView)에서도 입력이 답답하지 않게 한다.

## 현재 구현 구조

현재 사주 입력 폼은 아래 파일에서 관리한다.

- 입력 폼: `src/components/k-saju/SajuForm.tsx`
- 생년월일 선택: `src/components/k-saju/BirthDateSelect.tsx`
- 개인정보 동의: `src/components/legal/PrivacyConsent.tsx`
- 결과 표시: `src/components/k-saju/SajuResult.tsx`
- API: `src/app/api/saju/basic/route.ts`
- 타입: `src/lib/saju/types.ts`
- 출생 시간 옵션: `src/lib/saju/birth-time-options.ts`
- 시간대 옵션: `src/lib/saju/timezone.ts`

현재 `/saju` 페이지는 사주 허브 페이지다.

- 페이지: `src/app/[locale]/saju/page.tsx`
- 허브 컴포넌트:
  - `src/components/k-saju/SajuHubHero.tsx`
  - `src/components/k-saju/SajuHubGrid.tsx`
  - `src/components/k-saju/SajuElementsStrip.tsx`
  - `src/components/k-saju/SajuHubCta.tsx`

실제 기본 사주 입력은 `SajuForm` 컴포넌트를 사용하는 흐름이다.

## 현재 입력 데이터

`SajuForm`에서 보내는 기본 요청 데이터는 다음과 같다.

```ts
{
  petName: string;
  species: "dog" | "cat";
  petGender: "male" | "female";
  birthDate: string;
  birthTime: string | null;
  birthTimeUnknown: boolean;
  timezone: string;
  locale: "ko" | "en";
  privacyConsent: boolean;
}
```

현재 UI 입력 항목:

- 언어
- 반려동물 종류: 강아지 / 고양이
- 이름
- 동물 성별: 수 / 암
- 생년월일
- 출생 시간
- 출생 지역 시간대
- 개인정보 동의

## 핵심 UX 원칙

### 1. 사주 입력을 어렵게 보이게 하지 않기

사주, 만세력, 시간대 같은 단어는 사용자에게 어렵게 느껴질 수 있다.

권장 표현:

- `우리 아이 생일 정보를 알려주세요`
- `출생 시간을 몰라도 괜찮아요`
- `출생 지역 기준 시간으로 계산해요`
- `결과는 입력 후 바로 확인할 수 있어요`

피해야 할 표현:

- `만세력 필수 입력`
- `정확한 시주 산출을 위해...`
- `오차 발생 가능`

### 2. 입력 순서 단순화

추천 순서:

1. 이름
2. 종류
3. 성별
4. 생년월일
5. 출생 시간
6. 출생 지역 시간대
7. 개인정보 동의
8. 사주 보기

현재 구현은 종류가 먼저 나오지만, 새 디자인에서는 이름을 먼저 배치하는 편이 감정적 진입이 좋다.

### 3. 출생 시간 모름을 적극 지원

현재 `birthTime` 기본값은 `unknown`이다.

디자인에서는 이 선택지를 단순한 드롭다운 옵션보다 더 명확하게 보여주는 것이 좋다.

추천 UI:

- 토글 또는 카드:
  - `출생 시간을 알아요`
  - `출생 시간을 몰라요`
- 모를 경우:
  - 시간 드롭다운 숨김 또는 비활성화
  - 안내문 표시: `시간을 몰라도 연·월·일 기준으로 기본 사주를 볼 수 있어요.`

## 추천 화면 구성

### 1. Hero Header

목적: 입력 화면의 목적을 부드럽게 설명한다.

추천 카피:

- 제목: `우리 아이 K-사주 보기`
- 보조문구: `이름과 생일 정보를 입력하면 성향, 오행 밸런스, 케어 포인트를 알려드려요.`
- 보조 안내: `출생 시간을 몰라도 결과를 볼 수 있어요.`

추천 비주얼:

- 따뜻한 카드형 히어로
- 별, 달, 발자국, 오행 컬러 포인트
- 너무 무속적이기보다 귀엽고 프리미엄한 느낌

### 2. Pet Identity Card

목적: 사용자가 반려동물 정보를 입력하는 첫 단계.

입력:

- 이름
- 종류
- 성별

추천 디자인:

- 이름 입력은 큰 카드의 메인 필드로 배치
- 종류 선택은 강아지/고양이 카드 버튼
- 성별은 수/암 pill 버튼

예시:

```text
이름
[ 모찌                 ]

종류
[ 🐶 강아지 ] [ 🐱 고양이 ]

성별
[ ♂ 수 ] [ ♀ 암 ]
```

### 3. Birth Info Card

목적: 생년월일과 출생 시간 입력.

입력:

- 연도
- 월
- 일
- 출생 시간

현재 생년월일은 `BirthDateSelect`에서 연/월/일 3개 select로 처리한다.

추천 개선:

- 모바일에서는 연/월/일 select가 너무 좁아지지 않게 한다.
- 연/월/일 라벨을 명확히 표시한다.
- 출생 시간은 지지 시간대 설명을 접을 수 있게 한다.

출생 시간 옵션:

- 모름
- 자시 23:30 ~ 01:30
- 축시 01:30 ~ 03:30
- 인시 03:30 ~ 05:30
- 묘시 05:30 ~ 07:30
- 진시 07:30 ~ 09:30
- 사시 09:30 ~ 11:30
- 오시 11:30 ~ 13:30
- 미시 13:30 ~ 15:30
- 신시 15:30 ~ 17:30
- 유시 17:30 ~ 19:30
- 술시 19:30 ~ 21:30
- 해시 21:30 ~ 23:30

### 4. Timezone Card

목적: 출생 지역 시간대를 정확히 선택하게 한다.

현재 기본값:

- 브라우저의 `Intl.DateTimeFormat().resolvedOptions().timeZone`
- 실패 시 `Asia/Seoul`

추천 UI:

- 기본 선택값을 보여준다.
- 간단한 설명을 붙인다.

추천 카피:

```text
출생 지역 시간대
태어난 지역 기준 시간으로 계산해요. 한국에서 태어났다면 Asia/Seoul을 선택하면 됩니다.
```

주의:

- 너무 많은 시간대가 한 번에 보이면 복잡하므로 주요 시간대를 상단에 배치하는 방식도 좋다.
- 현재 `COMMON_TIMEZONES`와 감지된 현재 timezone을 합쳐 보여준다.

### 5. Privacy Consent

목적: 개인정보 동의가 필요하다는 점을 부담 없이 안내한다.

현재 동의하지 않으면 에러:

- 한국어: `개인정보 동의가 필요합니다.`
- 영어: `Please agree to the privacy notice.`

추천 UI:

- CTA 버튼 바로 위에 배치
- 긴 법적 문구보다 요약 문구 먼저 표시
- 자세한 내용은 펼침 또는 링크

추천 카피:

```text
입력한 생일 정보는 사주 계산과 저장된 리포트 확인에 사용됩니다.
```

### 6. Submit CTA

현재 CTA:

- 한국어: `사주 보기`
- 영어: `Reveal their vibe`

추천 CTA:

- `우리 아이 사주 보기`
- `오행 밸런스 확인하기`
- `결과 바로 보기`

로딩 문구:

- 현재: `만세력 계산 중...`
- 추천: `우리 아이 기운을 읽고 있어요...`

CTA는 화면 하단 고정 버튼으로도 고려할 수 있다.

모바일 앱에서는 입력 폼이 길어지므로, 사용자가 동의까지 완료하면 하단에 sticky CTA를 보여주는 방식이 좋다.

## 결과 이후 흐름

현재 결과가 나오면 아래 이어보기 섹션이 표시된다.

- 별자리 운세: `/saju/zodiac`
- 펫과 주인간 궁합: `/saju/compatibility`

연결 시 query string으로 입력값을 전달한다.

```text
petName
species
petGender
birthDate
birthTime
timezone
locale
```

추천 디자인:

- 결과 카드 아래 `이 정보로 이어보기` 섹션 유지
- 버튼 2개:
  - `별자리 운세보기`
  - `펫과 주인간 궁합보기`
- 로그인 필요 시 `/login`으로 이동

## 모바일 레이아웃 권장안

모바일 순서:

1. 짧은 Hero
2. 이름 입력
3. 종류 선택
4. 성별 선택
5. 생년월일 선택
6. 출생 시간 모름/알아요 선택
7. 시간대 선택
8. 개인정보 동의
9. 하단 CTA

모바일 주의점:

- 연/월/일 select는 최소 터치 높이 44px 이상
- 카드 간 간격은 충분히 둔다.
- 드롭다운이 많으므로 한 화면에 너무 많은 선택 UI를 몰아넣지 않는다.
- CTA는 명확히 하나만 강조한다.

## 데스크톱 레이아웃 권장안

데스크톱은 2컬럼 구성이 좋다.

왼쪽:

- Hero 설명
- 오행 미리보기
- 결과 예시

오른쪽:

- 입력 폼 카드

예시:

```text
[ 우리 아이 K-사주 설명 / 오행 미리보기 ] [ 입력 폼 카드 ]
```

폼 카드는 스크롤 중에도 보이도록 sticky 처리할 수 있다.

## 유지해야 할 기능

- 개인정보 동의 없이는 요청 불가
- 생년월일은 필수
- 이름은 필수, 최대 32자
- 출생 시간 모름 지원
- 시간대 선택 지원
- 결과 후 별자리/궁합으로 이어보기
- 로그인 사용자라면 Supabase 저장 흐름 유지
- 비로그인/게스트 사용자도 기본 결과 확인 가능

## 완료 기준

리뉴얼 완료 기준:

- `/ko/saju` 또는 사주 입력 진입 화면에서 입력 플로우가 명확하다.
- 모바일에서 모든 입력 필드가 터치하기 쉽다.
- 출생 시간을 모르는 사용자가 막히지 않는다.
- 개인정보 동의 누락 시 에러가 명확하다.
- `POST /api/saju/basic` 요청 데이터가 기존 타입과 호환된다.
- 결과 표시와 이어보기 링크가 기존처럼 동작한다.
- 한국어/영어 모두 깨지지 않는다.
- 기존 별자리/궁합/프리미엄 페이지에 영향이 없다.
